# IMaugrenI

Local-first tooling and method repositories for inspectable systems, clear boundaries, and small public building blocks.

## Product line

- **Observe** — [`tof_container_pulse`](https://github.com/IMaugrenI/tof_container_pulse): local Docker host status at a glance
- **Ground** — [`tof_local_knowledge`](https://github.com/IMaugrenI/tof_local_knowledge): local document indexing, search, and grounded answers
- **Generate** — [`tof_local_builder`](https://github.com/IMaugrenI/tof_local_builder): controlled local AI-assisted build workspace
- **Structure** — [`local_case_organizer`](https://github.com/IMaugrenI/local_case_organizer): local dossier organization, timelines, and export packages

These repositories are public tooling and method artifacts. They are meant to be readable, bounded, and useful on their own.

## Method repositories

- [`tof-showcase`](https://github.com/IMaugrenI/tof-showcase) — public overview of the repo family
- [`tof-v7-public-frame`](https://github.com/IMaugrenI/tof-v7-public-frame) — reduced public boundary frame
- [`tof-module-ideas`](https://github.com/IMaugrenI/tof-module-ideas) — module candidates and early framing
- [`tof-legacy-recovery-workbench`](https://github.com/IMaugrenI/tof-legacy-recovery-workbench) — public-safe recovery workflow baseline
- [`tof-bridge-planning-method`](https://github.com/IMaugrenI/tof-bridge-planning-method) — bridge-planning from prepared blocks to target candidates

## Principles

- local-first over cloud-first where possible
- explicit boundaries between roles and spaces
- inspectable outputs over opaque automation
- small, focused repositories over merged super-tools
