# Security Policy

## Supported Versions

Security fixes are considered for:

- the current `main` branch
- the latest public release, where a repository uses releases

Older snapshots may not receive updates.

## Reporting a Vulnerability

Please do not open public GitHub issues for security-sensitive reports.

Use GitHub private vulnerability reporting where available. If that is not available for the affected repository, contact the maintainer directly and include:

- the affected repository name
- a short description of the issue
- reproduction steps or proof of concept, if safe to share
- impact and any suggested mitigation

Do not post secrets, tokens, credentials, private keys, or sensitive sample data in public issues, pull requests, or discussions.

A response time is not guaranteed. Reports will be reviewed as time allows, and validated issues will be addressed in a way that fits the repository's public scope and maintenance capacity.
